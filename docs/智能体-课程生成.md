# 课程生成流程处理
根据之前的实践经验，我整理了完整的课程生成流程，包括从准备到完成的所有步骤：

## 一、准备阶段
### 1. 规则学习
- 仔细阅读《课程生成规则.md》，了解文件命名、文档结构、内容组织等要求
- 确认课程的基本信息字段、章节字段和课时字段的具体要求
### 2. 目录结构规划
- 在 docs 目录下创建独立的课程目录，如 primary_poetry_courses
- 为每个年级创建单独的课程文件，使用 global_小学X年级古诗词学习课程.md 命名格式
### 3. 内容准备
- 确定课程的整体结构，包括章节划分和课时安排
- 为每个课时准备详细的学习内容，包括诗歌原文、作者介绍、诗歌解析、学习目标、教学步骤和练习题目
- 确保内容适合目标年级学生的认知水平
## 二、分类管理
### 1. 获取现有分类
- 使用 mcp_edu-stdio_get_all_categories 工具获取系统中已有的分类
- 检查是否存在适合古诗词课程的分类
### 2. 创建分类
- 如果不存在合适的分类，使用 mcp_edu-stdio_create_category 工具创建主分类，如 "语文文学"
- 创建子分类，如 "古诗词"，用于存放古诗词学习课程
## 三、课程创建
### 1. 创建课程基本信息
- 使用 mcp_edu-stdio_create_course 工具创建课程
- 填写课程标题、描述、价格、时长、难度、状态等基本信息
- 设置课程的开始和结束时间，使用正确的ISO时间格式
### 2. 创建章节
- 根据课程结构，使用 mcp_edu-stdio_create_chapter 工具创建各个章节
- 为每个章节设置标题、描述和排序顺序
### 3. 创建课时
- 为每个章节创建相应的课时，使用 mcp_edu-stdio_create_lesson 工具
- 填写课时标题、描述、内容、时长、是否免费等信息
- 确保课时内容详细，包含完整的学习资料
## 四、内容管理
### 1. 课时内容编写
- 为每个课时编写详细的markdown内容，包括：
  - 诗歌原文
  - 作者介绍
  - 诗歌解析
  - 学习目标
  - 教学步骤
  - 练习题目
- 确保内容符合markdown格式规范，结构清晰
### 2. 内容审核
- 检查课程内容是否符合课程生成规则
- 确保内容适合目标年级学生的认知水平
- 验证所有字段填写正确，无遗漏
## 五、测试与完善
### 1. 课程预览
- 检查课程的整体结构和内容是否完整
- 验证章节和课时的排序是否正确
- 确保所有链接和资源都能正常访问
### 2. 内容调整
- 根据预览结果，对课程内容进行必要的调整和完善
- 确保课程内容生动有趣，符合学生的学习需求
## 六、发布与管理
### 1. 课程发布
- 将课程状态设置为 "published"，确保课程可以正常访问
- 确认课程的所有信息都已正确填写
### 2. 后续管理
- 定期更新课程内容，保持内容的新鲜度和相关性
- 收集学生的反馈，不断改进课程质量
- 为后续年级的课程创建积累经验
## 七、技术要点
### 1. 工具使用
- 熟练使用 MCP 工具进行分类、课程、章节和课时的创建
- 注意工具参数的正确格式，特别是时间格式
- 确保所有请求都能成功执行
### 2. 内容质量
- 确保课程内容符合教育要求，具有教育价值
- 注重内容的科学性、准确性和趣味性
- 保持内容的一致性和连贯性
### 3. 流程规范
- 严格按照课程生成规则进行操作
- 遵循先分类、后课程、再章节、最后课时的创建顺序
- 确保每个环节都有详细的记录和文档
通过以上流程，可以系统、高效地生成符合要求的古诗词学习课程，为学生提供优质的学习资源。

# 课程生成规则

## 1. 文件命名规则
- **全局文档**: 使用 `global_` 前缀
- **文件名格式**: `global_课程标题.md`
- **保存位置**: 项目的 `docs` 目录下

## 2. 文档结构规则

### 2.1 课程基本信息
- 课程ID
- 课程标题
- 课程描述（涵盖的科目和目标）
- 课程价格（包含原价）
- 课程时长（小时和分钟两种单位）
- 课程难度
- 课程状态（已发布/published）
- 创建时间

### 2.2 课程详情
- 按科目分类（如语文、数学、英语）
- 每个科目下列出具体学习内容

### 2.3 课程特色
- 列出课程的独特优势和特点

### 2.4 章节与课时详情
- 每个科目作为一个章节
- 每个章节包含章节ID和章节描述
- 每个章节下包含多个课时
- 每个课时包含：
  - 课时标题和课时ID
  - 类型: markdown（固定值）
  - 时长（分钟）
  - 是否免费（是/否）
  - 详细内容描述（markdown格式）

### 2.5 课程统计
- 总章节数
- 总课时数
- 免费课时数
- 总课程时长

### 2.6 创建记录
- 创建时间
- 创建方式（使用 edu-stdio MCP 工具）
- 服务前缀（service-edt-plat）

## 3. 内容组织规则
- **科目覆盖**: 必须包含主要科目
- **免费课时设置**: 每个科目设置一个免费课时（作为试看内容）
- **课时数量**: 每个科目包含4个课时
- **内容深度**: 针对目标学生群体的认知水平
- **内容类型**: 以markdown为主，包含知识点讲解、技巧训练、专项练习等

## 4. 课时内容规则
- **类型**: 固定为 "markdown"
- **内容格式**: 使用markdown语法，支持标题、列表、强调等格式
- **内容要求**: 
  - 知识点讲解清晰
  - 实例丰富
  - 练习题目适量
  - 学习目标明确
  - 生成的内容包含全部的信息，不是概括文字

## 5. 服务标识规则
- 服务前缀: service-edt-plat
- 创建方式: 使用 edu-stdio MCP 工具

## 7. 数据字段说明

### 7.1 课程基本信息字段
- **课程ID**: 唯一标识，数字类型
- **课程标题**: 课程名称，清晰明了
- **课程描述**: 课程概述，包含目标和内容范围
- **课程价格**: 当前售价，格式为 ¥XXX
- **原价**: 原售价，格式为 ¥XXX
- **课程时长**: 总课时长度，同时提供小时和分钟
- **课程难度**: 初级/中级/高级
- **课程状态**: 已发布（published）
- **创建时间**: ISO格式日期

### 7.2 章节字段
- **章节ID**: 唯一标识，数字类型
- **章节标题**: 章节名称
- **章节描述**: 章节内容概述
- **排序顺序**: 章节在课程中的顺序

### 7.3 课时字段
- **课时ID**: 唯一标识，数字类型
- **课时标题**: 课时名称
- **类型**: 固定为 "markdown"
- **时长**: 课时长度（分钟）
- **是否免费**: 是/否
- **内容**: 详细学习内容（markdown格式）
- **排序顺序**: 课时在章节中的顺序

## 8. 注意事项
- 所有修改记录都保存在项目的docs目录下
- 每一个文档添加服务的前缀，全局文档使用golbal_前缀
- 课时类型必须使用markdown类型
- 内容应保持专业、准确、易理解
- 遵循markdown语法规范，确保格式正确
