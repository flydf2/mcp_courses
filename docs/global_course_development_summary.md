# 课程开发与AI生成规则总结

## 一、开发过程

### 1. 项目结构
- **服务目录**：`/System/Volumes/Data/webcode/mcp_courses/service-edt-plat/`
  - **工具模块**：`tools/` 目录包含课程、章节、课时的创建工具
  - **客户端模块**：`client/` 目录包含与外部API交互的客户端
  - **配置模块**：`config/` 目录包含服务配置文件
  - **主入口**：`main.go` 负责服务启动和工具注册

### 2. 核心功能实现
- **课程创建**：`CourseTool` 结构体及其 `handleCreateCourse` 方法
- **章节创建**：`ChapterTool` 结构体及其方法
- **课时创建**：`LessonTool` 结构体及其方法
- **模拟响应**：为了便于测试，实现了模拟API响应的功能

### 3. 技术栈
- **编程语言**：Go
- **MCP服务**：基于Model Context Protocol实现
- **服务模式**：支持stdio、http、sse三种模式
- **配置管理**：环境变量和配置文件

### 4. 开发流程
1. **需求分析**：理解课程创建的业务逻辑和数据结构
2. **代码实现**：编写工具结构体和处理方法
3. **配置调整**：设置服务模式和相关参数
4. **测试验证**：通过模拟响应验证功能是否正常
5. **文档更新**：维护课程文档和生成规则

## 二、使用访问

### 1. 服务启动
- **配置文件**：修改 `config/config.env` 中的 `SERVICE_MODE` 为 `stdio`
- **启动命令**：在 `service-edt-plat` 目录下运行 `go run main.go`

### 2. MCP工具调用
- **课程创建**：使用 `mcp_edu-stdio_create_course` 工具
- **章节创建**：使用 `mcp_edu-stdio_create_chapter` 工具
- **课时创建**：使用 `mcp_edu-stdio_create_lesson` 工具

### 3. 数据传递
- **课程数据**：按照规则构建完整的课程信息JSON
- **章节数据**：包含章节标题、描述和排序
- **课时数据**：包含课时标题、描述、时长、类型等信息

### 4. 响应处理
- 服务返回JSON格式的响应，包含状态码、消息和数据
- 成功创建时返回相应的ID和创建信息

## 三、AI生成课程的规则

### 1. 数据结构要求
- **课程结构**：包含标题、描述、详情、价格、分类等字段
- **章节结构**：包含标题、描述、排序和课时列表
- **课时结构**：包含标题、描述、时长、类型等字段

### 2. 字段生成规则
- **课程字段**：
  - `title`：2-50个字符，简洁明了
  - `description`：50-200个字符，简要描述
  - `price`：1-9999，合理价格
  - `category_id`：有效的分类ID
  - `level`：beginner, intermediate, advanced
  - `status`：默认"published"
  - 数组字段：课程亮点、教学方法、支持服务、职业前景

- **章节字段**：
  - `title`：5-30个字符，反映章节内容
  - `description`：30-100个字符
  - `sort_order`：从1开始递增
  - `lessons`：每个章节至少1个课时

- **课时字段**：
  - `title`：5-30个字符
  - `description`：20-80个字符
  - `duration`：300-3600秒
  - `sort_order`：从1开始递增
  - `is_free`：默认false
  - `type`：video, document, text

### 3. 数据验证规则
- **必填字段检查**：确保所有必填字段都有值
- **数据类型检查**：确保字段类型正确
- **长度检查**：确保字符串字段长度符合要求
- **范围检查**：确保数值字段在合理范围内
- **格式检查**：确保日期、URL等格式正确
- **分类ID检查**：确保category_id在系统中存在
- **逻辑检查**：确保original_price >= price

### 4. 内容质量要求
- **准确性**：课程内容描述准确，符合实际
- **一致性**：课程、章节、课时内容相互一致
- **完整性**：信息完整，无缺失重要内容
- **专业性**：语言专业，符合教育领域规范
- **吸引力**：内容有吸引力，能够激发学习兴趣

### 5. 生成流程
1. **分析输入**：理解用户需求和课程主题
2. **生成课程信息**：根据规则生成课程基本信息
3. **生成章节结构**：设计合理的章节划分
4. **生成课时内容**：为每个章节生成具体课时
5. **数据验证**：检查生成的数据是否符合规则
6. **格式化输出**：按照API要求的格式输出数据

### 6. 注意事项
- 所有创建的课程默认教师ID为1
- 课程状态默认为"published"
- 数组类型的字段如果为空，会自动转换为空数组
- 确保提供的分类ID在系统中存在
- 生成的内容应符合教育领域的专业标准
- 避免生成重复或无关的内容
- 确保生成的数据结构与API接口要求一致

## 四、实际应用示例

以"小学六年级全科学习课程"为例：
1. **课程创建**：使用规则生成完整的课程信息，包含标题、描述、价格等
2. **章节创建**：创建语文、数学、英语三个章节
3. **课时创建**：为每个章节创建4个课时，包含免费和付费内容
4. **文档更新**：将生成的课程ID、章节ID、课时ID更新到文档中
5. **验证检查**：确保所有数据符合规则要求

通过这套完整的流程，我们成功创建了符合要求的小学六年级全科学习课程，包含了语文、数学、英语三个学科的系统内容，为学生提供了全面的学习资源。
