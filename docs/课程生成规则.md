# 课程生成规则

## 1. 文件命名规则
- **全局文档**: 使用 `global_` 前缀
- **文件名格式**: `global_课程标题.md`
- **保存位置**: 项目的 `docs` 目录下

## 2. 文档结构规则

### 2.1 课程基本信息
- 课程ID
- 课程标题
- 课程描述（涵盖的科目和目标）
- 课程价格（包含原价）
- 课程时长（小时和分钟两种单位）
- 课程难度
- 课程状态（已发布/published）
- 创建时间

### 2.2 课程详情
- 按科目分类（如语文、数学、英语）
- 每个科目下列出具体学习内容

### 2.3 课程特色
- 列出课程的独特优势和特点

### 2.4 章节与课时详情
- 每个科目作为一个章节
- 每个章节包含章节ID和章节描述
- 每个章节下包含多个课时
- 每个课时包含：
  - 课时标题和课时ID
  - 类型: markdown（固定值）
  - 时长（分钟）
  - 是否免费（是/否）
  - 详细内容描述（markdown格式）

### 2.5 课程统计
- 总章节数
- 总课时数
- 免费课时数
- 总课程时长

### 2.6 创建记录
- 创建时间
- 创建方式（使用 edu-stdio MCP 工具）
- 服务前缀（service-edt-plat）

## 3. 内容组织规则
- **科目覆盖**: 必须包含主要科目
- **免费课时设置**: 每个科目设置一个免费课时（作为试看内容）
- **课时数量**: 每个科目包含4个课时
- **内容深度**: 针对目标学生群体的认知水平
- **内容类型**: 以markdown为主，包含知识点讲解、技巧训练、专项练习等

## 4. 课时内容规则
- **类型**: 固定为 "markdown"
- **内容格式**: 使用markdown语法，支持标题、列表、强调等格式
- **内容要求**: 
  - 知识点讲解清晰
  - 实例丰富
  - 练习题目适量
  - 学习目标明确
  - 生成的内容包含全部的信息，不是概括文字

## 5. 服务标识规则
- 服务前缀: service-edt-plat
- 创建方式: 使用 edu-stdio MCP 工具

## 7. 数据字段说明

### 7.1 课程基本信息字段
- **课程ID**: 唯一标识，数字类型
- **课程标题**: 课程名称，清晰明了
- **课程描述**: 课程概述，包含目标和内容范围
- **课程价格**: 当前售价，格式为 ¥XXX
- **原价**: 原售价，格式为 ¥XXX
- **课程时长**: 总课时长度，同时提供小时和分钟
- **课程难度**: 初级/中级/高级
- **课程状态**: 已发布（published）
- **创建时间**: ISO格式日期

### 7.2 章节字段
- **章节ID**: 唯一标识，数字类型
- **章节标题**: 章节名称
- **章节描述**: 章节内容概述
- **排序顺序**: 章节在课程中的顺序

### 7.3 课时字段
- **课时ID**: 唯一标识，数字类型
- **课时标题**: 课时名称
- **类型**: 固定为 "markdown"
- **时长**: 课时长度（分钟）
- **是否免费**: 是/否
- **内容**: 详细学习内容（markdown格式）
- **排序顺序**: 课时在章节中的顺序

## 8. 注意事项
- 所有修改记录都保存在项目的docs目录下
- 每一个文档添加服务的前缀，全局文档使用golbal_前缀
- 课时类型必须使用markdown类型
- 内容应保持专业、准确、易理解
- 遵循markdown语法规范，确保格式正确
